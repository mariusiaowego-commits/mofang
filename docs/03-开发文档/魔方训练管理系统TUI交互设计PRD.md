## 1. 设计理念

### 1.1 核心设计原则

> **快、静、准** - 这就是速拧的精神，也是这个TUI的精神。

| 原则 | 说明 |
|------|------|
| **键盘优先** | 100%纯键盘操作，不用碰鼠标 |
| **零依赖** | 只用Python标准库，pip install什么都不需要 |
| **极速响应** | 打开即用，启动 < 0.5秒 |
| **无干扰** | 训练时界面简洁，只有关键信息 |
| **数据可视化** | ASCII图表，直观展示进步 |

### 1.2 颜色方案

魔方主题配色（暗色系，护眼）：

| 颜色 | 用途 | ANSI码 |
|------|------|--------|
| 白色 | 正文，普通文本 | 0 |
| 黄色 | 高亮，PB，最佳成绩 | 3 |
| 绿色 | 成功，达标，进步 | 2 |
| 红色 | 警告，DNF | 1 |
| 蓝色 | 标题，边框 | 4 |
| 青色 | 提示，快捷键 | 6 |
| 灰色 | 次要信息，占位符 | 7 |

### 1.3 快捷键体系

**全局通用快捷键：**

| 按键 | 功能 |
|------|------|
| `ESC` / `q` | 返回上一级 / 退出 |
| `j` / `↓` | 向下移动 |
| `k` / `↑` | 向上移动 |
| `Enter` | 确认 / 进入 |
| `h` / `?` | 帮助弹窗 |
| `1-9` | 快速跳转对应菜单项 |

## 3.3 P2 - 训练计时页（核心功能！）

**功能：**
- 大字体实时计时器
- WCA官方打乱公式显示
- 实时统计面板（当前5次的AO5）

- DNF / +2 惩罚标记

**界面效果：**
```

║                                                     ║
║  打乱公式:                                          ║
║  R U R' U R U2 R' F R U R' U' F'                    ║
║                                                     ║
║  ┌───────────────────────────────────────────────┐ ║
║  │                                               │ ║
║  │           00:01.234                           │ ║
║  │                                               │ ║
║  │      [空格开始/停止]  [R重置]  [D删除]        │ ║
║  └───────────────────────────────────────────────┘ ║
║                                                     ║
║  📊 当前组统计                                       ║

║  #  时间      日期       备注                      ║
║  ───────────────────────────────────────           ║
║  1   82.3s    16:05                               ║
║  2   78.5s    16:04       🔥 PB!                   ║
║  3   91.2s    16:03                               ║
║  4   DNF      16:02       角块没归位               ║
║  5   87.1s    16:01       +2                       ║
║                                                     ║
╠═════════════════════════════════════════════════════╣
║  空格: 计时  R:重置  D:删除  N:新打乱  ESC:返回     ║
╚═════════════════════════════════════════════════════╝
```

**核心交互逻辑：**

| 状态 | 空格行为 |
|------|---------|
| 等待中 | 开始计时 → 开始计时状态 |
| 计时中 | 停止计时 → 保存记录 → 自动生成新打乱 |
| 已停止 | 生成新打乱 → 等待状态 |

**快捷键：**
- `空格`：开始/停止计时
- `R`：重置当前计时
- `D`：删除最后一条记录
- `N`：新打乱公式
- `+`：给最后一条加 +2 惩罚
- `F2`：标记为 DNF
- `ESC`：返回仪表盘

## 4. 技术实现方案

### 4.1 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| **TUI库** | Python 标准库 `curses` | 零依赖，跨平台 |
| **数据存储** | JSON 文件 | 简单易读，无需数据库 |
| **打乱算法** | 内置纯Python实现 | 符合WCA标准，零依赖 |
| **图表** | 纯ASCII字符绘制 | 无需绘图库 |

### 4.2 项目文件结构

```
mofang/
├── scripts/
│   ├── cubetrainer/           # TUI主程序目录
│   │   ├── __init__.py
│   │   ├── app.py            # 主应用入口
│   │   ├── ui/               # UI组件
│   │   │   ├── components.py # 通用组件（按钮、卡片、列表）
│   │   │   ├── pages.py      # 各页面实现
│   │   │   └── theme.py      # 颜色主题
│   │   ├── core/             # 核心逻辑
│   │   │   ├── timer.py      # 计时器
│   │   │   ├── stats.py      # 统计计算
│   │   │   └── scramble.py   # 打乱公式生成
│   │   └── data/             # 数据层
│   │       └── models.py     # 数据模型
│   └── cubetrain.py          # 启动脚本
├── training/
│   └── users/                # 用户数据目录
│       └── 用户1.json
└── docs/03-开发文档/         # 本文档
```
